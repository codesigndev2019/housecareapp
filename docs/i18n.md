# i18n Guidelines

We recommend using `@ngx-translate/core` for runtime language switching and lazy-loading translation JSON files.

## Structure
- `src/assets/i18n/{es.json,en.json}` for root translations.
- Prefer keys grouped by feature: `auth.login.title`, `auth.register.submit`, etc.
- Use `Intl` API for dates/numbers and register locale data via `registerLocaleData` when needed.

## Adding a new language
1. Add `xx.json` to `src/assets/i18n/` with the key-value pairs.
2. Add fallback logic in `I18nService` and test switching at runtime.
